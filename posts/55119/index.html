写在前面

在我们平常工作中经常会遇到有关于文件的相关处理操作，相关的API也有不同，如果条件受限，那么我们可以使用jdk自带的Files类; 如果又引入apache依赖，就可以使用FileUtiles工具类。

File类

作用

  • 获取路径（绝对路径和相对路径）
  • 判断文件是否存在
  • 创建、删除文件

获取路径

  • getAbsolutePath() ：返回此File的绝对路径。
  • getPath()：结果和getAbsolutePath一致。
  • getName()：返回文件名或目录名。
  • length()：返回文件长度，以字节为单位。

File对象表示文件的时候，length()才会有具体值，表文件夹则无值

判断文件是否存在

  • exists() ：判断文件或目录是否存在。
  • isDirectory() ：判断是否为目录。
  • isFile() ：判断是否为文件。

创建、删除文件

  • createNewFile()：文件不存在，创建一个新的空文件并返回true，文件存在，不创建文件并返回false。
  • delete()：删除文件或目录。
  • mkdir()：创建目录。
  • mkdirs() ：创建目录，包括任何必需但不存在的父目录。

一个File对象代表硬盘中实际存在的一个文件或者目录

delete方法，如果此File表示目录，则目录必须为空才能删除。

mkdir()只能创建一级目录，mkdirs()可以创建多级目录，开发中遇到过此问题，建议使用mkdirs()

Paths类

作用

将路径字符串或在连接时形成路径字符串的字符串序列转换为路径

需要传入一个绝对路径作为参数

转换为路径

  • get()
  • startsWith(String path) : 判断是否以 path 路径开始
  • endsWith(String path) : 判断是否以 path 路径结束
  • isAbsolute(): 判断是否是绝对路径
  • of()：通过转换路径字符串或连接后形成路径字符串的字符串序列，返回一个路径

Files类

Java NIO Files类（java.nio.file.Files）操作文件（file）和文件夹（Directory）的工具类。

Files类通常是和Paths类一起使用的

作用

Java NIO2中增加了Files工具类封装提供了一些操作文件系统文件的工具方法。

判断文件是否存在

  • exists()
  • notExists()

删除文件

  • delete()
  • deleteIfExists()

删除文件或者空文件夹，如果文件夹不为空的情况下会报错；建议使用deleteIfExists()

创建文件及文件夹

  • createDirectories()
  • createFile()

写文件

  • write()

举个荔枝：

读文件

  • readAllLines()

举个荔枝：

拷贝文件

  • copy()

移动文件

  • move()

FileUtils

作用

FileUtils存在于Common工具库中。包含一些工具类,它们基于File对象工作，包括读，写，拷贝和比较文件。

maven依赖

拷贝文件

  • copyDirectory()：复制文件夹（文件夹里面的文件内容也会复制）
  • copyDirectoryToDirectory()：以子目录的形式将文件夹复制到到另一个文件夹下
  • copyFile()：复制文件

移动文件

  • moveDirectoryToDirectory()：以子文件夹的形式移动到另外一个文件下

清空、删除文件夹

  • deleteDirectory()：删除文件夹，包括文件夹和文件夹里面所有的文件
  • cleanDirectory()：清空文件夹里面的所有的内容

创建文件夹

  • forceMkdir()：创建文件夹
